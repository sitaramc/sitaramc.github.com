1 Gitolite book outline

1.1 Chapter 1: What is Gitolite? (pages: 6)

This chapter briefly describes what is gitolite and why you might need it. It will also show how to create a "throwaway" installation to play with.

  • Common Access Control needs
  • Access Control provided by git and the OS
  • Access Control example with gitolite
  • A selection of other gitolite "power" features
  • Trying out gitolite

1.2 Chapter 2: Installing Gitolite (pages: 5)

This chapter will cover a fresh installation from source.

  • Server pre-requisites
  • Getting the source
  • Running the install step
  • Running the setup step
  • Checking your new installation

By the end of this chapter, the user has a working installation of gitolite.

1.3 Chapter 3: Your users and gitolite (pages: 8)

This chapter will cover what you need to tell your users about the gitolite system you're managing and how they should use it.

  • Accessing Gitolite
  • Getting information from Gitolite
  • Running Gitolite commands
  • Getting help
  • Troubleshooting SSH issues

1.4 Chapter 4: Adding and removing users (pages: 8)

This chapter will show you how to manage users on your Gitolite system.

  • Adding users
  • Authentication versus Authorisation
  • Users with more than one pubkey
  • Giving some users a "shell"
  • Diagnosing pubkey problems

1.5 Chapter 5: Adding and removing repos (pages: 4)

This chapter will cover managing repositories on your Gitolite system.

  • Adding new repositories
  • Adding your existing repositories
  • Migrating repositories from older gitolite
  • Common problems and troubleshooting

1.6 Chapter 6: Basic Access Control (pages: 6)

This chapter will show how to let different users have different levels of access to the repositories on the system.

  • Basic access control examples
  • Syntax of the conf file
  • Gitolite access control rules
  • Branch level access control and refexes
  • User and repo groups
  • Rule accumulation

1.7 Chapter 7: Advanced Access Control and Configuration (pages: 10)

This chapter will talk about some advanced features of Gitolite that make the administrator's job easier in many ways.

  • Personal branches
  • Types of write operations
  • Gitweb and git-daemon
  • Applying "deny" rules during the pre-git access check
  • Git "config" and Gitolite "options"
  • Introduction to VREFs

1.8 Chapter 8: Allowing users to create their own repos (pages: 6)

This chapter will explain the widely used "wild repos" feature, with some common use cases and examples.

  • What are "wild" repos
  • Writing a wild repo specification
  • Roles
  • Explaining wild repos to your users
  • Managing an entire site with just wild repos!

1.9 Chapter 9: Customising Gitolite (pages: 4)

This chapter describes the "non-core" parts of Gitolite and what they do. It then goes into how you can customise these scripts or write your own.

  • "Core" and "non-core" Gitolite
  • Some shipped examples of non-core features
  • Customising non-core programs for your site
  • Writing your own non-core programs

1.10 Chapter 10: VREfs (pages: 6)

This chapter will delve into the powerful VREF feature, using several existing VREF scripts to illustrate. It will also show how to create your own VREFs with a few examples.

  • Adding additional update hooks
  • VREF concepts and terminology
  • Fallthru in VREF rules
  • Arguments passed to a VREF
  • Designing a VREF
  • Writing and testing VREFs

1.11 Chapter 11: Mirroring (pages: 8)

Gitolite's mirroring feature is very powerful, and very useful for large multi-site installations. This chapter will show how to set up mirroring with an example of 3 sites, and describe how to use gitolite mirroring optimally.

  • Gitolite mirroring concepts
  • Initial setup
  • HOSTNAME substitution and local repos
  • Allowing slaves to redirect pushes
  • Manual synchronisation
  • Switching to a different master
